# Phase 3 — Project Case Studies (Plan)

## Objective
Create the case study structure and draft content for featured projects so the portfolio can move from content architecture into publication-ready project pages.

## Scope
- Draft the case study template for each featured project.
- Synthesize project insights from the project documentation folders.
- Prepare a clear story structure: Overview, Problem, Architecture, Challenges, Trade-offs, Results.
- Identify required assets and content gaps for each project.

## Deliverables
1. Case study outline for each project:
   - myU
   - MyU ID
   - Hung Hau House
   - Events Platform
2. Optional secondary project outline:
   - Face Verification
3. Draft content blocks for each section.
4. Source references and notes to support future writing.

## Success Criteria
- Every featured project has a clear case study outline.
- Each outline contains enough detail to write the final case study page.
- The content is grounded in the existing project docs in `Docs/myu & id-myu`, `Docs/events-myu-vn`, and `Docs/Hung Hau House`.

## Suggested Workflow
1. Review existing project documents and extract core facts.
2. Draft project summary for each case study.
3. Build the story skeleton (Overview → Problem → Architecture → Challenges → Trade-offs → Results).
4. Mark content that still needs confirmation from Thien.

## Notes
- This phase focuses on project storytelling and case study structure.
- Visual assets and detailed design will be handled later.
