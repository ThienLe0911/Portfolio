# Phase 4 — Content Hierarchy

## Home (`/`)
1. Hero
2. About
3. Key Metrics (max 4)
4. Featured Projects (top 3-4)
5. Career Timeline Preview
6. Learning (compact)
7. Contact CTA

Hierarchy intent:
- Từ định vị -> bằng chứng -> dự án -> chuyển đổi.

## Projects Listing (`/projects`)
1. Page Intro (ngắn)
2. Filter/Category (nếu cần)
3. Project Cards
4. Optional secondary project block
5. CTA to Contact

Hierarchy intent:
- Ưu tiên scan nhanh và vào sâu từng case study.

## Project Detail (`/projects/{slug}`)
1. Project Header (name, role, stack, snapshot)
2. Overview
3. Problem
4. Architecture
5. Challenges
6. Trade-offs
7. Results
8. Learnings
9. Next Iteration
10. Related Project Links
11. Contact CTA

Hierarchy intent:
- Kể chuyện theo logic engineering decision, tránh kể lan man theo thời gian.

## Timeline (`/timeline`)
1. Intro
2. Milestone list (descending)
3. Cross-link sang project detail
4. Contact CTA

Hierarchy intent:
- Thể hiện progression và độ tích lũy kinh nghiệm.

## Contact (`/contact`)
1. Quick intro
2. Contact methods (email/LinkedIn/GitHub)
3. Resume download
4. Availability/status

Hierarchy intent:
- Trang chuyển đổi ngắn, rõ, ít nhiễu.

## Cross-page Priority Rules
- Primary info luôn nằm above-the-fold.
- Mỗi trang có tối đa 1 CTA chính.
- Không lặp nguyên văn nội dung giữa Home và Project Detail.
